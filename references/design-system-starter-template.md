# Fable Design System Starter Template

## Intake questions
1. Who is the primary user and what decision/action should the screen support?
2. Should we preserve the current brand, evolve it, or create a new visual language?
3. Which references should this feel close to, and which should it avoid?
4. Is the base system shadcn/Radix, Material, existing project tokens, or custom?
5. What charts/data views must the system support?
6. Does this need dark mode, accessibility/compliance, localization, or mobile-first constraints?

## Token checklist
- Color roles
- Typography roles
- Spacing/rhythm
- Radius/elevation
- Motion/easing/duration
- Component states
- Chart palette and data semantics
- Accessibility rules

## Follow-through checklist
- `DESIGN.md` or token file created/updated.
- `CLAUDE.md` tells agents to obey the design system.
- Components reference tokens, not random values.
- Charts use chart tokens and themed tooltip/legend/card components.
- Future screen instructions point back to the system.
